# Local Manga Browser

Local Manga Browser 是一款面向 Android 的本地漫画阅读器，适合已经把漫画下载到手机、平板或外置存储中的用户。它的重点不是在线浏览，而是帮你把本地漫画整理得更好找、更好读、更不容易乱。

如果你的漫画来自 EHViewer 或其他相似工具，通常会以“一个漫画一个文件夹”的形式保存在本地。这个 App 就是为这种使用方式准备的：选择漫画目录，扫描漫画，然后用自定义画廊、tag 搜索和阅读历史来管理自己的本地漫画库。

## 主要特色

### 本地阅读，不依赖云端

漫画文件保存在你自己的设备上。App 通过 Android 的系统目录选择器读取漫画目录，不需要把漫画上传到服务器，也不要求联网使用。

### 不污染漫画源目录

App 会区分“漫画目录”和“工作目录”：

- 漫画目录：只放你的漫画文件，App 只读取，不写入。
- 工作目录：保存 App 自己产生的数据，例如画廊、tag、阅读历史、上次阅读页等。

这样即使你以后换 App、迁移漫画、重新整理文件夹，原始漫画目录也能保持干净。

### 自定义画廊

你可以按自己的习惯创建画廊，例如：

- 默认画廊
- 待看
- 已整理
- 喜欢的作者
- 特定主题或分类

首页会显示当前所在画廊。点击“选择画廊”后，会进入画廊选择页面。每个画廊会像漫画卡片一样展示，封面使用该画廊中的第一本漫画；如果画廊里还没有漫画，会显示“无漫画”。

### Tag 标记与搜索

每本漫画都可以添加 tag。tag 可以用来标记作者、题材、语言、状态或任何你想记录的信息。

搜索支持简单但实用的规则：

- 输入 `A B`：查找同时包含 A 和 B 的漫画。
- 输入 `A -B`：查找包含 A，但不包含 B 的漫画。

例如：

```text
中文 纯爱
```

表示查找同时带有“中文”和“纯爱”tag 的漫画。

```text
中文 -未整理
```

表示查找带有“中文”，但不带“未整理”的漫画。

### 封面提前缓存

App 会在后台提前加载漫画封面。主页、画廊选择页、漫画详情页、历史记录页、搜索页等位置都会复用同一套封面缓存，减少临时加载时的空白和闪烁。

封面默认使用漫画的第一页。如果第一页是 GIF，封面也会按 GIF 显示。

### 阅读历史与续读

App 会记录你读过哪些漫画、上次读到第几页、最近阅读时间。

历史记录页会按最近阅读时间排序，点击历史中的漫画可以直接进入阅读器，并恢复到上次离开的页数。

### 沉浸式阅读

阅读器界面会尽量把屏幕留给漫画本身，不显示顶部标题栏和返回按钮。

阅读时支持：

- 点击左半屏上一页
- 点击右半屏下一页
- 左右滑动翻页
- 滑动翻页时页面跟随手指移动
- 双指缩放
- 缩放后拖动查看图片
- 点击底部区域显示阅读进度条
- 拖动进度条快速跳页

## 推荐的漫画文件夹结构

当前推荐使用“一个漫画一个文件夹”的结构：

```text
漫画根目录/
├── 漫画 A/
│   ├── 001.jpg
│   ├── 002.jpg
│   ├── 003.jpg
│   └── ...
├── 漫画 B/
│   ├── 01.png
│   ├── 02.png
│   ├── 03.png
│   └── ...
└── 漫画 C/
    ├── page_001.webp
    ├── page_002.webp
    └── ...
```

每个子文件夹会被识别为一本漫画。文件夹名通常会作为漫画标题。

支持常见图片格式：

```text
jpg, jpeg, png, webp, gif, bmp, avif
```

### 文件命名建议

为了保证页面顺序正确，建议图片文件名带有清晰的页码：

```text
001.jpg
002.jpg
003.jpg
```

或：

```text
page_001.png
page_002.png
page_003.png
```

不建议混用很乱的命名，例如：

```text
1.jpg
10.jpg
2.jpg
cover.jpg
final-new.png
```

如果文件名数字顺序清晰，阅读体验会更稳定。

### EHViewer 类漫画目录

如果你的漫画是从 EHViewer 或类似 App 保存的，只要它最终是类似下面的结构，一般就适合扫描：

```text
下载目录/
├── [作者] 漫画标题 1/
│   ├── 0001.jpg
│   ├── 0002.jpg
│   └── ...
├── [作者] 漫画标题 2/
│   ├── 0001.jpg
│   ├── 0002.jpg
│   └── ...
└── ...
```

## 首次使用方法

1. 安装并打开 App。
2. 进入设置页。
3. 点击选择漫画目录，选择保存漫画的根目录。
4. 点击选择工作目录，选择一个专门给 App 保存数据的目录。
5. 点击扫描漫画。
6. 回到首页查看扫描出的漫画。

建议漫画目录和工作目录不要互相包含。例如：

```text
Pictures/Manga/          # 漫画目录
Documents/MangaReader/   # 工作目录
```

不要这样放：

```text
Pictures/Manga/
└── AppWorkspace/
```

也不要这样放：

```text
Documents/MangaReader/
└── Manga/
```

这样做是为了避免 App 状态文件和漫画文件混在一起。

## 日常使用

### 选择画廊

首页会显示当前画廊。点击“选择画廊”进入画廊选择页，然后选择：

- 全部画廊
- 某个自定义画廊

选择后会回到首页，并只显示该画廊中的漫画。

### 管理画廊

在画廊相关页面可以进入画廊管理，进行：

- 新建画廊
- 重命名画廊
- 删除画廊

删除画廊时，原本属于该画廊的漫画会迁移到默认画廊，不会删除漫画文件。

### 给漫画添加 Tag

进入漫画详情页后，可以添加或删除 tag。tag 只保存在 App 的数据中，不会写入漫画文件夹。

### 搜索漫画

进入搜索页后输入 tag 条件：

```text
tag1 tag2 -tag3
```

表示同时包含 `tag1` 和 `tag2`，并排除 `tag3`。

当前搜索不做“或者”匹配，主要用于精确筛选。

### 阅读漫画

进入详情页后点击阅读。

阅读器操作：

- 点左侧：上一页
- 点右侧：下一页
- 左右滑动：翻页
- 双指捏合：缩放
- 缩放后单指拖动：移动图片
- 点底部区域：显示或隐藏进度条
- 拖动进度条：跳转页数

## 安装与本地测试

如果你只是想使用 App，安装构建好的 APK 即可。

如果你想在本地运行项目，可以使用：

```powershell
npm.cmd install
npm.cmd run server
```

然后在浏览器打开：

```text
http://127.0.0.1:4173
```

浏览器预览主要用于查看界面和调试数据流程。真实的 Android 目录授权、本地图片读取、滑动翻页、缩放拖动等体验，建议安装到 Android 真机或模拟器测试。

## 打包 Android APK

```powershell
npm.cmd run android:sync
cd android
$env:JAVA_HOME='C:\Program Files\Android\Android Studio\jbr'
$env:PATH="$env:JAVA_HOME\bin;$env:PATH"
.\gradlew.bat assembleDebug
```

生成的 debug APK 位于：

```text
android/app/build/outputs/apk/debug/app-debug.apk
```

连接 Android 设备后可以安装：

```powershell
adb install -r android\app\build\outputs\apk\debug\app-debug.apk
```

## 当前说明

- 当前重点支持图片文件夹形式的本地漫画。
- 压缩包格式可能会被识别为条目，但 Android 端暂未作为主要阅读格式支持。
- 大体积漫画或超高分辨率图片会占用更多内存。
- App 不会主动删除你的漫画源文件。
